Авторизация: База данных PsSql поля имя пользователя пароль. С одним именем пользователя - двух записей быть не должно. При первом вводе происходит регистрация.

В проект вложены две папки каталогов:
klient file - хранятся клиентские файлы
server file / имя пользователя - хранит пользовательские файлы на сервере 

Описание исполнения механизма команд:
Всего 4 команды:
1) Удаление файла на клиенте. Просто удаляем файлы в папке klient file.
2) Добавление файла на клиенте. Просто добавляем файл в папку klient file на клиенте.
3) Скачать на сервер. Файл из папки klient file отправляем на сервер. С начало отправляется команда содержащая информацию о файле, потом сам файл.
4) Синхронизация на клиенте. Все файлы из папки клиента отправляем на сервер. С начало отправляется команда на сервер с требованием удалить все файлы на сервере, сервер их
удаляет и отправляет результат об успешном удалении. Дальше клиент по одному файлу на сервер. Перед отправкой следующего файла клиент ждет ответа от сервера об успешном принятии
преведущего файла.
5) Скачать на клиента. Скачивает файл с сервера на клиент.
6) Удалить файл с сервера. С клиента передается имя файла который удаляется на сервере.
7) Синхронизация на сервере. переносит все файлы с сервера на клиент.



Для работы программы нужна бд pgSQL. Архив в папке с проектом. Описание структуры базы класс Users в модуле Server/
